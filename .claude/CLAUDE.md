# CLAUDE.md

## 0. 目的
このプロジェクトで Claude Code が作業する際の「言語・ブラウザ操作・Git 運用・参照ドキュメント」を明示する。

## 1. 言語ポリシー（必須）
- すべての対話・出力は **日本語** で行うこと。
- ただし **Git のコミットメッセージは英語**で記述すること（後述の規約に従う）。

## 2. ブラウザ操作（必須）
- Web ブラウズ／UI 自動化が必要な作業は **MCP の Playwright** のみを使用すること。
- `fetch` / `curl` など **他の手段によるネットワークアクセスは原則禁止**。必要な場合は理由を説明し、Playwright MCP で代替できないかを検討する。

## 3. Git 運用（必須）
  ### 基本的なpush運用
  - 「push して」という指示を受けた場合は、以下の順序で実施すること :
    1. 変更点の確認（`git status` / `git diff`）
    2. ステージング（`git add`）・ すでにステージングされているものがある場合は新たにステージングしない
    3. コミット（英語・Conventional Commits 準拠を推奨。例：`feat: ...`, `fix: ...`, `docs: ...`）
    4. リモートへのプッシュ（デフォルトブランチに直 push が不適切な場合は PR 作成を提案）

  ### Pull Request作成の自動化
  - 「pr作成して」という指示を受けた場合は、以下の手順を自動実行すること :
    1. 現在の変更状況とブランチ構成を確認
    2. 現在のブランチから新しい作業用ブランチを作成（命名規則：`fix/`, `feat/`, `style/`等の接頭辞を使用）
    3. 変更をコミット（Conventional Commits準拠の英語メッセージ）
    4. 新しいブランチをリモートにプッシュ
    5. 元のブランチに対してPull Requestを作成（日本語の詳細な説明付き）
    6. 元のブランチに戻る
    7. 作業用ブランチをローカル・リモートともに削除してクリーンアップ

  ### コミットメッセージ規約
  - コミットメッセージは **要約（50 文字程度） + 必要ならボディ**。自動生成時も英文の簡潔さを優先。
  - Pull Requestの説明は日本語で詳細に記述すること。

## 4. セーフティガード
- ファイル編集・依存追加・外部通信は、プロジェクトの既定ルールに従うこと。危険操作は事前に差し戻す。
- ブラウザ自動化の対象サイトは最小限に限定し、個人情報や秘密情報を扱わない。

## 5. 実行時チェック
- 起動後 `/status` でワークスペース/権限を確認し、必要に応じて設定を提案する。

## 6. 多角的な検討/修正
必要に応じて、あるいは指示を受けた場合は、codex MCPを実行して、他のAIモデルの意見を参考にすること。
codex MCPで使用するモデルは、`gpt-5-codex`またはそれに準ずるモデルとする。
例えば以下の場合にcodex MCPを実行すること。ただし、通常時はClaude単独で作業を完結させること。
- バグ修正が3回以上失敗した場合、codex MCPに処理を委譲する
  - Claudeの役割
    初期リクエストの処理、失敗時のCodex MCPへのエスカレーション
    ClaudeはCodex MCPに対して、失敗の詳細とこれまでの試行内容を提供し、問題解決のための具体的な指示を与える
  - Codex MCPの役割
    Claudeからのフォールバックリクエストを受け、より高度なコード生成を実行
- 極めて複雑なタスクをサブタスクに分解し、各分野に特化したエージェントに委任する
  - Claudeの役割
    タスク全体の計画立案、サブタスクへの分解、Codex MCPへのコード生成タスクの委任、結果の統合
  - Codex MCPの役割
    Claudeからの具体的なコード生成/編集命令を受け、専門的なワーカーとして高品質なコードを生成
- 新しい機能を追加、あるいは重要な機能を編集/修正する際に慎重な検討が必要な場合、codex MCPに意見を求める。
  同じタスクを複数の異なるエージェントに検討させ、結果を比較・統合することで品質と信頼性を向上させる。
- Claudeが作成したコードに対して、codex MCPにレビューを依頼し、改善点や潜在的なバグを指摘してもらう。
  ただし、Codex MCPのレビューはあくまで参考意見として扱い、最終的な判断はClaudeが行う
  - Claudeの役割
    コードの初期生成、Codex MCPへのレビュー依頼、レビュー結果の受け取りと必要に応じて反映
  - Codex MCPの役割
    Claudeから提供されたコードを詳細にレビューし、改善点やバグを指摘
